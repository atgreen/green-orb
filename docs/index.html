Green Orb

An Observe and Report Buddy for your SRE toolbox

Get Started Download GitHub

Wrap your app, watch logs, act fast. Green Orb turns patterns into actions—notify, webhooks, Kafka, or scripts—without code changes. Ideal for containers, on‑call speed, and pragmatic SRE automation.

Why Green Orb?

Green Orb tails your app’s stdout/stderr and turns regex matches into actions—send notifications, fire webhooks, publish to Kafka, run scripts (e.g., thread dumps), suppress output, or restart/kill—no code changes, just a tiny sidecar process.

Smart Notifications

Send alerts to Slack, Email, Discord, Teams, and more.

Webhooks & Kafka

Call HTTP endpoints or publish to Kafka topics.

Automated Actions

Run shell scripts on matches; use capture groups.

Process Control

Restart or kill the observed process via signals.

Built-in Metrics

Prometheus counters, histograms, and gauges.

Periodic Checks

HTTP/TCP checks + flapping detection.

Quick Start

  1. Download the orb binary from Releases.
  2. Create a config file named green-orb.yaml in your working directory.
  3. Run your app through orb:
$ orb run <your command>
# Example
$ orb run java -jar mywebapp.jar

Using Docker? Set your ENTRYPOINT to run via orb:

ENTRYPOINT ["orb", "run", "java", "-jar", "app.jar"]

Minimal Example

channels:
  - name: "startup-email"
    type: "notify"
    url:  "smtp://USER:PASS@smtp.gmail.com:587/?from=USER@gmail.com&to=USER@gmail.com&subject=App%20Starting!"

signals:
  - regex: "Starting Application"
    channel: "startup-email"

Reference regex capture groups via environment variables ORB_MATCH_1, ORB_MATCH_2, … in exec channels.
